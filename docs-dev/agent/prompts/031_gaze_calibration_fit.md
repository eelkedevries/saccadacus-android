# Task: Gaze calibration — capture and affine fit

## Goal

Add a guided calibration that captures the active gaze signal at known on-screen targets and
fits an affine map to normalised screen coordinates, then persists it.

## Scope

Implement only the work described in this prompt. Do not implement adjacent systems or future prompts. This is the calibration screen + capture + affine fit + persistence + unit tests for the fit. It does **not** apply the calibration to the live signal or CSV (that is prompt 032).

## Context

Spec (prompt 029) scopes in calibration: an affine map fitted from on-screen targets producing
a normalised point-of-gaze, subsuming `SignConvention`. The active gaze signal comes from the
eye-local pipeline (iris or blendshape, prompt 030). Settings persistence uses `AppSettings`
(prompt 025); the unit-test infra exists (prompts 006 / 028).

## Required changes

1. A calibration screen that shows N targets (e.g. 5 or 9) at known normalised screen
   positions; for each, prompt the user to look at it and capture the averaged gaze signal over
   a short dwell.
2. Fit a 2-D **affine** map (`screen_x = a·gx + b·gy + c`, and likewise for `screen_y`) by
   least squares from the captured (gaze → target) pairs; store the coefficients in a
   `Calibration` holder.
3. Persist the calibration via `AppSettings`/DataStore, and add JUnit tests for the affine fit
   (points generated by a known affine map are recovered within tolerance; a degenerate/under-
   determined input is handled without crashing).

## Do not implement

Do not implement:
- applying the calibration to the live signal, overlay, or CSV (prompt 032);
- validation / error metrics (prompt 033);
- non-linear / homography models.

## Acceptance criteria

The task is complete when:
- `./gradlew assembleDebug` builds and `./gradlew testDebugUnitTest` passes in CI (incl. the fit tests);
- running calibration captures the targets and stores a fitted, persisted mapping.
